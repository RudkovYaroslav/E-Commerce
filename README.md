# Анализ продаж в E-Commerce

## Обзор проекта
В рамках данного проекта проводится анализ данных о продажах в  E-Commerce с целью оценки эффективности бизнеса и выявления изменений в поведении клиентов за два сопоставимых периода.

Анализ сосредоточен на ключевых показателях, таких как выручка, количество заказов и средняя стоимость заказа, с целью понимания изменений в моделях покупок.

---

## Набор данных
Набор данных содержит транзакционные данные электронной коммерции, включая:

- order_id
- order_date
- region
- revenue

Данные представляют собой транзакции продаж и были получены из [Kaggle](https://www.kaggle.com/datasets/miadul/e-commerce-sales-transactions-dataset?resource=download).

---

## Инструменты и технологии
- SQL (PostgreSQL) — извлечение данных, агрегирование и расчет показателей  
- Python (Pandas, Seaborn) — анализ и визуализация данных  

---

## Рабочий процесс анализа
1. Проверка качества данных (дубликаты, отсутствующие значения, анализ распределения)  
2. Расчет ключевых показателей (выручка, количество заказов, средняя и медианная стоимость заказа)  
3. Сравнение двух периодов (январь–август 2024 г. и январь–август 2025 г.)  
4. Региональный анализ  
5. Визуализация трендов в Python  

---

## Основные выводы
После корректировки с учетом периодов с неполными данными анализ показывает, что выручка выросла примерно на 1%, а количество заказов — примерно на 3%.

Это указывает на снижение средней стоимости заказа. В то же время медиана стоимости заказа осталась стабильной, что свидетельствует о том, что поведение большинства клиентов существенно не изменилось.

В целом результаты указывают на изменение моделей поведения покупателей, сопровождающееся снижением доли заказов с высокой стоимостью.
